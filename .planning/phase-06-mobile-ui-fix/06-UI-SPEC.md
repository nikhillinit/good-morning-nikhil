## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gap (e.g., ShowBadge emoji + name) |
| sm | 8px | Input internal gap, chip grid gap |
| md | 16px | Content card internal padding (horizontal), px-4 base |
| lg | 24px | Content card internal padding (vertical), section stacking gap |
| xl | 32px | Card-to-edge margin on mobile (max-w-md + auto margins) |
| 2xl | 48px | Minimum touch target height — ALL interactive elements |
| 3xl | 64px | Bottom safe-area clearance for back button / bottom-anchored controls |

Exceptions:
- Touch targets: ALL interactive controls (buttons, chips, checkboxes, MuteToggle, SkipButton, back button) must have `min-h-[48px]` and `min-w-[48px]` where applicable. This is non-negotiable per Apple HIG 44pt and WCAG 2.5.5 (AAA target 44×44, AA floor 24×24).
- MuteToggle: currently `h-10 w-10` (40px). Must be upgraded to `h-12 w-12` (48px).
- ShowBadge `top-4 left-4`: 16px from safe area. Acceptable; do not change padding.
- Caption band: `bottom-24` (96px) from bottom — keep as-is; audit did not flag caption positioning.

## Typography

| Role | Size | Weight | Line Height | Tailwind class | Notes |
|------|------|--------|-------------|----------------|-------|
| Body | 16px | 400 | 1.5 | `text-base` | Default prose, used in `ContinueButton` description |
| Label | 14px | 400 | 1.4 | `text-sm` | Input labels (`TwoText`), chip label instructions, checkbox label — differentiated from body by size (14px vs 16px) and color |
| Question Prompt | 16px | 400 | 1.5 | `text-base italic` | `QuestionPrompt` — UPGRADE from current `text-sm text-zinc-400` |
| Heading / Display | 24px+ | 700 | 1.2 | `font-display text-2xl` | Show titles, `StartButton`, `SubmitButton` |

**Critical change:** `QuestionPrompt` currently uses `text-sm text-zinc-400` — this is the root contrast failure identified as Critical (audit 0:05–2:12). Must change to `text-base text-white` (or `text-zinc-100`) inside a content card.

**MadLib body stem text:** currently `text-base text-zinc-300` — upgrade to `text-base text-white`.

**MultiSelect / SingleSelect label:** currently `text-sm text-zinc-400` — upgrade to `text-sm text-zinc-200`.

**InvestOrPass helper text:** currently `text-xs text-zinc-500` — upgrade to `text-sm text-zinc-300`.

**skipBtn:** currently `text-sm text-zinc-500` — upgrade to `text-sm text-zinc-400 hover:text-zinc-200`.

**Minimum contrast ratios enforced:**
- Normal text (below 18px): WCAG 4.5:1 minimum
- Large text (18px bold or 24px+ regular): WCAG 3:1 minimum
- All UI state text on card background (`#1c1c1f`): must pass 4.5:1

## Color

### Global Token Additions (add to `globals.css` `:root`)

```css
--card: #1c1c1f;           /* content card background — replaces floating-text-on-overlay pattern */
--card-border: rgba(255,255,255,0.08);  /* subtle card edge separation */
--prompt: #ffffff;         /* question prompt foreground — replaces zinc-400 */
--input-border: rgba(255,255,255,0.18);  /* input border at rest — replaces zinc-700 (too dark) */
--input-border-focus: #eab308;  /* existing accent — keep */
--placeholder: #71717a;    /* zinc-500 — contrast-evaluated against card bg (#1c1c1f): ~3.2:1 for large hint text. Use with font-size >= 16px only. */
--chip-idle-border: rgba(255,255,255,0.20);  /* replaces zinc-700 border on chips */
--chip-idle-text: #d4d4d8;   /* zinc-300 — 7.5:1 on card bg. PASS. */
--nav-control: rgba(0,0,0,0.70);  /* ShowBadge, SkipButton, MuteToggle backdrop */
```

### Color Contract

| Role | Value | Usage | Ratio on bg |
|------|-------|-------|-------------|
| Dominant (60%) | `#000000` (bg image + overlay) | Full-screen theatrical background | — |
| Secondary (30%) | `#1c1c1f` (`--card`) | Content card behind all interactive UI (QuestionPrompt + UIInput) | — |
| Accent (10%) | `#eab308` (`--accent`, yellow-500) | Primary CTA buttons, focus ring, selected chip border, MadLib underline, selected state highlight | — |
| Destructive | `#f87171` (`--error`) | Inline validation error text only — no destructive actions in this phase | — |

Accent reserved for: primary CTA button fill (`bg-yellow-500`), keyboard focus ring (`outline: 2px solid var(--accent)`), selected chip border (`border-yellow-500`), selected chip highlight tint (`bg-yellow-500/20`), MadLib inline input underline (`border-b-2 border-yellow-500`).

**NOT accent:** back button, SkipButton, MuteToggle, ShowBadge, caption band. These use the nav-control dark-glass treatment.

### Content Card Pattern (NEW — PRIMARY AUDIT FIX)

All interactive content (QuestionPrompt + UIInput) must render inside a content card:

```
bg-[--card] / rounded-2xl / px-5 py-6 / w-full max-w-md
border border-[--card-border] / backdrop-blur-sm
```

This replaces the floating-text-on-overlay pattern that caused the contrast failures across every screen. The theatrical background remains fully visible around the card edges — the card is a foreground container, not a modal overlay.

Card sits in the `main` z-10 content layer, centered vertically. On screens with only a `start-button` or `submit-button` (no form content), the card is omitted — those buttons stand alone against the background.

ARIA: Content card div gets `role="region" aria-label="Survey question"` so screen readers announce the interactive area. Inputs already have associated labels or placeholders in code.

### Input Field Tokens (replacing current `inputField` constant)

```
border border-[--input-border]   ← was: border-zinc-700 (too low contrast)
bg-zinc-900/90                   ← was: bg-zinc-900/80 (slightly more opaque for readability)
text-white                       ← keep
placeholder:text-zinc-500        ← keep; only acceptable at 16px+ input size
focus:border-[--input-border-focus]  ← keep accent focus
focus:outline-none               ← keep
rounded-lg px-4 py-3            ← keep
```

**Contrast check:** `#71717a` (zinc-500) on `#18181b` (zinc-900): ~3.0:1. Acceptable for placeholder text (hint role, not primary content). Against card `#1c1c1f`: ~2.9:1 — marginal. Mitigated by input field having its own `bg-zinc-900/90` background, so placeholder contrast is evaluated against the input bg, not the card.

### Chip State Tokens (replacing `chipIdle` / `chipSelected`)

| State | Border | Background | Text | Contrast |
|-------|--------|------------|------|---------|
| Idle | `rgba(255,255,255,0.20)` | `bg-zinc-900/80` | `text-zinc-300` (#d4d4d8) | 7.5:1 on zinc-900 — PASS |
| Selected | `border-yellow-500` | `bg-yellow-500/20` | `text-yellow-300` (#fde047) | 8.4:1 on zinc-900 tint — PASS |
| Disabled (not used) | — | — | — | — |

### Back Button Redesign

Current: `bg-white/10 text-white/70` — fails contrast (white/70 ≈ 4.0:1 — marginal) and nearly invisible background.

New contract:
```
bg-black/70 text-white rounded-full px-4
min-h-[48px] flex items-center gap-2
backdrop-blur-sm border border-white/10
hover:bg-black/80
```

Text: `text-white` (full opacity) — 21:1 contrast on black/70 bg — PASS.

Position: `absolute left-4 bottom-8 safe-bottom z-20` — keep existing positioning.

### Navigation Controls (ShowBadge, SkipButton, MuteToggle)

All use `bg-[--nav-control]` = `rgba(0,0,0,0.70)` with `text-white` at full opacity. Currently ShowBadge uses `bg-black/60` — upgrade to `bg-black/70`.

MuteToggle: upgrade from `text-white/70` to `text-white`. Upgrade from `h-10 w-10` to `h-12 w-12`.

SkipButton: upgrade from `text-white` (already full opacity — keep) background from `bg-white/10` to `bg-black/70`.

**Nav stacking contract (locked):**
- MuteToggle: `fixed top-4 right-4 z-30` — always visible, highest z
- SkipButton: `absolute top-16 right-4 z-10` — below MuteToggle, only visible during audio playback
- ShowBadge: `absolute top-4 left-4` — opposite corner, no collision

## Content Card Layout Contract

The content card is the single most impactful fix from the audit. Implementation rule:

```
When screen.ui is NOT 'none' AND NOT 'start-button' AND NOT 'submit-button':
  Wrap <QuestionPrompt> + <UIInput> in a content card div
  Card: bg-[--card] rounded-2xl px-5 py-6 w-full max-w-md border border-[--card-border] backdrop-blur-sm

When screen.ui IS 'start-button' OR 'submit-button':
  Button stands alone (no card) — theatrical presentation preserved
  Button itself already has sufficient contrast (dark text on yellow, or yellow on dark)

When screen.ui IS 'continue-button':
  Wrap in card — the descriptive body text in ContinueButton is currently low contrast
```

The card must NOT cover the full viewport. It sits as a floating card in the center-bottom area of the screen, leaving the theatrical background visible above it.

## Mobile Viewport Contract

- Primary target: 375px wide (iPhone SE / standard mobile)
- All content cards: `w-full max-w-md mx-auto` — 448px max, full-width on mobile
- No horizontal scroll permitted
- Keyboard active: `body.keyboard-active .fixed.bottom-0 { display: none }` already in globals.css. Content card must NOT be `fixed bottom-0` — it must be in normal flow inside `relative z-10` main so it scrolls with keyboard.
- Keyboard CTA: When keyboard is active, the primary CTA button ("Lock it in") must be `sticky bottom-0` within the content card's scroll area, so it's always reachable without scrolling past inputs. Use `sticky bottom-0 z-10` on the button wrapper inside the card.
- Orientation: content card uses `overflow-y-auto max-h-[70svh]` to handle landscape mode gracefully
- Safe areas: `safe-top` and `safe-bottom` classes already in globals.css — apply `safe-bottom` to back button positioning
